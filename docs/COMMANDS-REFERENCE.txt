--- LOAD TEST FROM AMAZON LINUX EC2 (recommended — same VPC as EMQX) ---
WHEN  : Run load from EC2 instead of your laptop (more connections, stable network)
WHERE : Your PC once (print env) + Amazon Linux EC2 in ap-south-1 (same VPC or reachable NLB)

On your PC (project root) — copy/paste output to EC2:
  bash ./scripts/print_load_test_env.sh

On Amazon Linux EC2 (after git clone this repo, or scp/rsync project folder):
  cd emqx
  bash ./scripts/setup_loadgen_amazon_linux.sh          # once: python3, venv, pip deps
  bash ./scripts/run_load_test_on_ec2.sh probe          # quick MQTT check
  bash ./scripts/run_load_test_on_ec2.sh staged        # ~15 min autoscaling demo
  bash ./scripts/run_load_test_on_ec2.sh 2k            # orchestrated 2K dashboard demo (~30 min)
  CLIENTS=100 bash ./scripts/run_load_test_on_ec2.sh sustained   # until Ctrl+C

Manual env (if print_load_test_env.sh unavailable):
  export AWS_REGION=ap-south-1
  export MQTT_HOST=$(terraform output -raw mqtt_nlb_dns_name)   # run on PC, paste on EC2
  export ASG_NAME=$(terraform output -raw replicant_asg_name)

EC2 requirements:
  - Amazon Linux 2 or 2023 (or Ubuntu) in same region
  - Security group: outbound to NLB :1883 allowed (default usually OK)
  - IAM role with autoscaling:DescribeAutoScalingGroups optional (ASG polling in staged_load.py)

Launch a small load-generator (AWS Console):
  EC2 -> Launch instance -> Amazon Linux 2023 -> t3.small -> same VPC/subnet as EMQX
  -> SSH in -> sudo dnf install -y git -> git clone <your-repo> -> follow steps above

================================================================================
E) AWS CLI COMMANDS (TROUBLESHOOTING + SCREENSHOTS)
================================================================================
WHERE : Git Bash or any terminal with aws CLI

--- ASG capacity (during load test) ---
  aws autoscaling describe-auto-scaling-groups \
    --region ap-south-1 \
    --auto-scaling-group-names emqx-prod-replicants-asg \
    --query "AutoScalingGroups[0].DesiredCapacity" --output text
Expected: 1 = idle, 2+ = scaled out

--- ASG table view ---
  aws autoscaling describe-auto-scaling-groups \
    --region ap-south-1 \
    --auto-scaling-group-names emqx-prod-replicants-asg \
    --query "AutoScalingGroups[0].{Desired:DesiredCapacity,Min:MinSize,Max:MaxSize,Instances:length(Instances)}" \
    --output table

--- NLB target health ---
  aws elbv2 describe-target-health \
    --region ap-south-1 \
    --target-group-arn "$(aws elbv2 describe-target-groups --region ap-south-1 --names emqx-prod-mqtt-tg --query 'TargetGroups[0].TargetGroupArn' --output text)" \
    --query "TargetHealthDescriptions[*].[Target.Id,TargetHealth.State]" --output table
Expected: healthy

--- CloudWatch alarms (why scale did not trigger) ---
  aws cloudwatch describe-alarms \
    --region ap-south-1 \
    --alarm-names emqx-prod-replicants-high-network emqx-prod-replicants-high-nlb-traffic emqx-prod-replicants-high-cpu emqx-prod-replicants-low-cpu \
    --query "MetricAlarms[*].[AlarmName,StateValue]" --output table

--- Instance refresh (after userdata / bootstrap script change) ---
  aws autoscaling start-instance-refresh \
    --region ap-south-1 \
    --auto-scaling-group-name emqx-prod-replicants-asg \
    --preferences MinHealthyPercentage=50,InstanceWarmup=600

AWS Console screenshots:
  EC2 -> Auto Scaling Groups -> emqx-prod-replicants-asg -> Group details + Activity
  EC2 -> Target Groups -> emqx-prod-mqtt-tg -> Targets tab

================================================================================

WHERE : SSH on EC2 core (section F)

  sudo grep PERF /var/log/emqx-bootstrap.log | tail -20

  grep -E 'MAX_PORTS|MAX_CONNECTIONS|ACCEPTORS|DIST_BUFFER|MQTT__MAX' /etc/emqx/terraform.env
  cat /etc/sysctl.d/99-emqx-performance.conf | head -15
  cat /etc/security/limits.d/99-emqx.conf
  swapon --show

Expected: PERF lines OK, fs.file-max=2097152, swap empty, LimitNOFILE=2097152

Terraform tuning vars (project root terraform.tfvars — optional):
  emqx_tune_nofile              = 2097152
  emqx_tune_max_ports           = 2097152
  emqx_tune_acceptors           = 64
  emqx_tune_max_connections     = 1024000
  emqx_tune_dist_buffer_size_kb = 2097151

After changing tuning: run instance refresh (section E) or terraform apply + replace instances.

Reference: https://docs.emqx.com/en/emqx/latest/performance/tune.html
